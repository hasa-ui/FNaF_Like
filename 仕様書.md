# Night Shift Survival 仕様書

## 1. 文書の目的
この文書は、2026-03-24 時点の実装を基準に、ブラウザ試作版 `Night Shift Survival` の現行仕様を整理したものである。
将来構想や拡張案は別文書 `ロードマップ.md` に分離し、本書では「今どう動いているか」を優先して記述する。

## 2. ゲーム概要

### 2-1. 作品概要
- 深夜勤務中の警備室を守り、`12 AM` から `6 AM` まで生存することを目的としたブラウザ用ホラーゲーム。
- プレイヤーは監視カメラ、左右ドア、電力を管理しながら敵の侵入を防ぐ。
- 1 プレイ完結型の単一 Night 試作であり、継続進行やセーブ要素は未実装。

### 2-2. ジャンル
- ホラー
- サバイバル
- リソース管理
- 監視 / 防衛ゲーム

### 2-3. 対応環境
- 主対象: デスクトップブラウザ
- モバイルブラウザ: レイアウト調整はあるが、操作密度の都合で補助対応扱い
- 依存ライブラリ: なし

### 2-4. ファイル構成
- `index.html`
- `style.css`
- `game.js`

静的ファイルのみで構成され、ビルド不要でローカル実行できる。

## 4. ゲーム進行

### 4-1. 状態遷移
ゲーム状態は以下の 5 種類を持つ。

- `title`
- `howto`
- `playing`
- `win`
- `lose`

`title` と `howto` はメニュー状態であり、プレイ用操作とゲーム進行ループは停止する。

### 4-2. 開始フロー
- 初期表示はタイトル画面
- `How To Play` から操作説明画面へ遷移可能
- `Start` でプレイ開始
- `Restart Shift` はタイトルへ戻らず、その場でプレイ状態を再初期化する

### 4-3. 時間進行
- プレイ開始時刻は `12 AM`
- 1 秒ごとに `minutesElapsed` が `1` 増加する
- `10` 経過ごとに表示時刻が 1 時間進む
- `minutesElapsed >= 60` で `6 AM` 到達となり勝利する

実プレイ時間は約 60 秒。

### 4-4. 勝利条件
- `6 AM` に到達すること

勝利時は以下を実行する。

- 状態を `win` へ変更
- 勝利パネル表示
- 勝利音再生
- 環境音停止

### 4-5. 敗北条件
以下のいずれかで敗北する。

- 左入口側の敵が到達した状態で左ドアが開いている
- 右入口側の敵が到達した状態で右ドアが開いている
- 停電中に `office` / `hallL` / `hallR` のいずれかへ敵がいる

敗北時は以下を実行する。

- 状態を `lose` へ変更
- ジャンプスケア演出を再生
- その後に敗北パネルを表示
- 環境音停止

## 5. プレイヤー操作

### 5-1. 操作一覧
- 左ドアの開閉
- 右ドアの開閉
- モニターの上げ下げ
- カメラ切替
- ゲーム開始
- How To Play 表示
- タイトルへ戻る
- シフト再開
- 音声オン / オフ切替
- 日本語 / 英語切替

### 5-2. 入力制御
- `playing` 以外ではプレイ操作を無効化
- モニター展開中は左右ドア操作を無効化
- カメラ切替演出中は入力をロック
- 勝敗確定後はプレイ操作を無効化

### 5-3. モニター操作の遅延
- モニター開閉時に約 `340ms` の入力ロックを入れる
- 実際の開閉状態反映は約 `160ms` 後
- 開閉時に専用 SE と画面フリップ演出を再生する

## 6. リソース管理

### 6-1. 電力
- 初期値は `100%`
- 1 秒ごとに減少
- 下限は `0%`

### 6-2. 使用量ルール
基本使用量は `1`。以下を加算する。

- 左ドア閉鎖中: `+1`
- 右ドア閉鎖中: `+1`
- モニター展開中: `+1`

実際の減少量は `0.42 × 使用量 / 秒`。

### 6-3. 停電時挙動
電力が `0%` になると以下を即時実行する。

- 左右ドアを強制的に開く
- モニターを強制的に閉じる
- ステータスを停電文言へ変更
- 停電 SE を 1 回のみ再生

その後、敵が `office` / `hallL` / `hallR` のいずれかにいれば、約 `2200ms` 後に敗北する。

## 7. カメラシステム

### 7-1. 監視可能カメラ
- `stage`
- `hallL`
- `hallR`
- `storage`
- `office`

### 7-2. カメラの基本仕様
- モニター展開中のみカメラ映像を表示する
- カメラボタンはモニター展開中のみ有効
- カメラ名と説明文は言語に応じて切り替わる

### 7-3. 現状の役割
現状のカメラは以下を担う。

- 敵位置の確認
- 画面演出
- 危険方向の補助判断

試作段階では、ゲームデザイン上の中核というより索敵手段としての比重が大きい。

## 8. 敵仕様

### 8-1. 登場敵
- `Rust`
- `Moth`
- `Grin`

### 8-2. 初期配置
- `Rust`: `stage`
- `Moth`: `stage`
- `Grin`: `storage`

### 8-3. ルート定義
- `Rust`: `stage -> office -> hallL`
- `Moth`: `stage -> storage -> hallR`
- `Grin`: `storage -> office -> hallL -> hallR`

ただし `office` 到達後の挙動は個別処理される。

- `Rust`: `office` から `hallL` へ移動
- `Moth`: `office` から `hallR` へ移動
- `Grin`: `office` から `hallL` / `hallR` のいずれかへランダム移動

そのため実プレイでは、入口付近と `office` を往復する形で圧力をかける。

### 8-4. 移動更新
- 3 秒ごとに敵移動判定を実行する
- 各敵はその時点の移動確率に基づいて前進する
- `hallL` / `hallR` に新規到達した敵がいれば接近 SE を再生する

### 8-5. AI 初期値
- `Rust`: `0.24`
- `Moth`: `0.18`
- `Grin`: `0.14`

### 8-6. AI 上昇量
3 秒ごとの更新時に以下を加算する。

- `Rust`: `+0.015`、上限 `0.92`
- `Moth`: `+0.013`、上限 `0.90`
- `Grin`: `+0.018`、上限 `0.94`

### 8-7. 個別特性
- `Rust`
  - `hallL` 以外では移動確率に `0.72` 倍補正がかかる
  - 左入口での侵入猶予は `550ms`
- `Moth`
  - モニター展開中は追加で移動圧が上がる
  - `mothAggro` を持ち、モニター使用で蓄積、未使用で減衰する
  - 右入口での侵入猶予は `900ms`
- `Grin`
  - `office` から左右いずれかへランダム移動する
  - Threat Readout 上で位置を安定表示しない場合がある
  - 左右入口での侵入猶予は `700ms`

### 8-8. Moth の活性化
`Moth` には専用の `mothAggro` がある。

- モニターを上げた瞬間に `+0.08`
- モニター展開中は毎秒 `+0.03`
- モニター非展開中は毎秒 `-0.05`
- 上限は `0.28`

さらにモニター展開中は移動確率へ `+0.06` の追加圧がかかる。

## 9. 脅威判定

### 9-1. 左側脅威
以下のいずれかが `hallL` にいると左側脅威とみなす。

- `Rust`
- `Grin`

左ドアが開いていれば、最短の侵入猶予後に敗北する。

### 9-2. 右側脅威
以下のいずれかが `hallR` にいると右側脅威とみなす。

- `Moth`
- `Grin`

右ドアが開いていれば、最短の侵入猶予後に敗北する。

### 9-3. Threat Readout
Threat Readout は右カラムに常時表示される補助情報である。

- モニターを下げている間は全敵を `UNKNOWN` / `不明` 表示
- モニター展開中は通常敵の現在位置を表示
- `Grin` は現在見ているカメラと一致しない場合、`UNSTABLE` / `不安定` 表示になる

現状では難易度を下げる補助機能として働いており、本番用 UI ではなく試作支援寄りの仕様である。

## 10. UI / 演出 / 音

### 10-1. ステータス表示
ステータス欄には以下を表示する。

- 現在状態に応じた主文言
- 危険度に応じたヒント文

一部操作時は一時的に短文ステータスへ切り替え、約 `1800ms` 後に通常の監視文言へ戻す。

### 10-2. ヒント表示の優先順位
ヒント文は概ね以下の優先順位で決まる。

1. タイトル / How To の案内
2. 電力危険域 (`15%` 以下)
3. Moth 活性化警告
4. 左右入口の危険
5. モニター使用中の注意
6. 異常なし

### 10-3. 言語切替
- 対応言語は日本語 / 英語
- 文言は `game.js` 内辞書で管理
- 以下が切替対象
  - タイトル / メニュー
  - 操作説明
  - HUD
  - ボタン
  - ステータス / ヒント
  - カメラ名 / 説明
  - 勝敗パネル

### 10-4. オーディオ
Web Audio API による簡易音響を実装している。

実装済みの音は以下。

- 環境音
- モニター上げ
- モニター下げ
- ドア操作
- 敵接近
- 停電
- ジャンプスケア
- 勝利

音声は上部ボタンでオン / オフ切替できる。
ブラウザが AudioContext を提供しない場合は無音で継続動作する。

### 10-5. ビジュアル演出
現行版には以下の演出を含む。

- オフィス背景
- カメラ画面ごとの差分背景
- 走査線 / ノイズ / ビネット
- カメラフリップ演出
- 入口脅威時の目の表示
- ジャンプスケアフラッシュ
- 勝敗オーバーレイ
- タイトル / How To オーバーレイ

## 11. 非機能仕様と制約

### 11-1. 実装上の制約
- ゲームロジック、UI、描画、音、文言辞書が `game.js` に集約されている
- データ定義はコード埋め込みで、外部設定ファイル化されていない
- テストコードや自動検証スクリプトは同梱していない

### 11-2. 未実装項目
以下は現時点で未実装である。

- Night 選択
- 難易度選択
- セーブ / ロード
- 設定保存
- 敵や施設の物語設定
- 敵ごとの専用ビジュアル / 専用音素材
- Threat Readout の難易度別制御

### 11-3. 現行版の位置づけ
現行版は「短時間で一連の流れを遊べる試作」であり、以下が成立している。

- 開始前導線
- プレイ本編
- 勝敗判定
- 最低限の音 / 演出
- 日英切替

一方で、ゲームの深さ、情報設計、拡張性、世界観表現は今後の開発対象とする。
